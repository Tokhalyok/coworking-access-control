# 🏢 Coworking Access Control

Система контроля доступа в коворкинг на основе блокчейн технологии.

## 📋 Описание проекта

Это приложение позволяет управлять доступом пользователей в коворкинг-пространство. Все действия (выдача доступа, отзыв, попытки входа) записываются в блокчейн и не могут быть изменены или удалены. Это обеспечивает прозрачность и безопасность системы.

### Что умеет приложение:
- Регистрировать пользователей с разными ролями (User, Admin)
- Выдавать и отзывать доступ
- Эмулировать замок — открывается при наличии доступа
- Вести неизменяемый журнал всех попыток входа в блокчейне
- Показывать историю посещений с временными метками

## 🚀 Запуск

### Шаг 1 — Запусти Ganache
Открой Ganache и убедись что работает на:
```
HTTP://127.0.0.1:8545
Network ID: 1337
```

### Шаг 2 — Задеплой контракт в Remix
1. Открой [remix.ethereum.org](https://remix.ethereum.org)
2. Вставь код из `CoworkingAccess.sol`
3. Скомпилируй (Ctrl+S)
4. Environment → Custom External Http Provider → `http://127.0.0.1:8545`
5. Нажми Deploy
6. Скопируй адрес контракта

### Шаг 3 — Настрой API.py
Вставь адрес контракта в `API.py`:
```python
contract_address = self.w3.to_checksum_address("ВАШ_АДРЕС_КОНТРАКТА")
```

### Шаг 4 — Скопируй ABI
В Remix → Solidity Compiler → кнопка ABI → скопируй → сохрани в `abi.json`

### Шаг 5 — Запусти приложение
```bash
python Run.py
```

## 📖 Как пользоваться

### Сценарий тестирования:

**1. Регистрация пользователя**
- Адрес Admin → первый адрес из Ganache
- Адрес нового пользователя → второй адрес из Ganache
- Имя → любое (например "Алибек")
- Роль → 0 — User
- Нажми **Зарегистрировать**

**2. Попытка входа без доступа**
- Вставь адрес пользователя в поле эмулятора
- Нажми **Войти в коворкинг**
- Результат: 🔒 Доступ запрещён

**3. Выдача доступа**
- Адрес Admin → первый адрес
- Адрес пользователя → второй адрес
- Нажми **✅ Выдать доступ**

**4. Вход с доступом**
- Снова нажми **Войти в коворкинг**
- Результат: 🔓 Замок открыт на 3 секунды

**5. Проверка журнала**
- Нажми **🔄 Обновить журнал**
- Видишь все попытки входа с временем и результатом

## 👥 Роли в системе

| Роль | Описание |
|------|----------|
| SuperAdmin | Создаётся при деплое. Управляет всем |
| Admin | Регистрирует пользователей, выдаёт/отзывает доступ |
| User | Обычный пользователь коворкинга |

---

## 🔒 Безопасность

- Все записи в блокчейне **неизменяемы** — никто не может удалить историю входов
- Только **Admin/SuperAdmin** может регистрировать пользователей и управлять доступом
- Валидация адресов — приложение проверяет корректность Ethereum адресов перед отправкой
